# Creating Bank Rules for Auto-Categorization

Bank rules are one of QuickBooks' most powerful time-saving features — once set up, they automatically categorize recurring transactions so you spend seconds on what used to take minutes.

## What You'll Learn

- What bank rules are and how they work behind the scenes
- How to create a new rule using Banking > Rules > New Rule
- How to set conditions based on vendor name, amount, and description text
- How to assign a default category and payee to matched transactions
- How rule priority affects which rule wins when multiple rules could apply
- How to edit, disable, or delete rules you no longer need

## What Bank Rules Are and Why They Save Hours

A bank rule is a set of instructions you give QuickBooks: "Whenever you see a transaction matching these conditions, automatically categorize it this way." Once a rule is in place, matching transactions bypass the manual review step and are either automatically categorized (ready to add with one click) or added to your books entirely on their own.

Think about a transaction like your monthly internet bill. Every month, Comcast charges your business account $129.99. Without a rule, you review it, confirm the category (Utilities: Internet), and click Add — every single month. With a rule, QuickBooks recognizes it, categorizes it as Utilities: Internet, and can even add it automatically without you touching it.

For businesses with 50–300 transactions per month, well-designed bank rules can cut active review time by half. Recurring vendors — subscriptions, utilities, suppliers you order from weekly, regular contractors — are all ideal candidates for rules.

## How to Create a New Bank Rule

Follow these steps to create your first rule:

1. From the left navigation menu, click **Banking** (or **Transactions**).
2. Click the **Rules** tab at the top of the Banking page.
3. Click the **New rule** button in the upper right.
4. In the **Rule name** field, enter a descriptive name (e.g., "Comcast Monthly Internet" or "Amazon — Office Supplies").
5. In the **Apply this rule to transactions that are** dropdown, select **Money out** (for expenses) or **Money in** (for income/deposits). Choose the correct direction first — this affects which conditions are available.
6. In the **And** dropdown, select which account(s) this rule should apply to (e.g., Business Checking, or all accounts).
7. Now set your **conditions**. You'll see a row with three dropdown fields: what to look at, how to compare it, and what value to match. (Details on conditions in the next section.)
8. Under **Then**, set what QuickBooks should do when the rule fires: assign a **Payee** name, assign a **Category** from your chart of accounts, and optionally set a **Class** or **Location** if you use those features.
9. Choose the **Action**: either **Categorize the transaction** (it still appears in For Review, pre-categorized, waiting for your click) or **Automatically add to my books** (it skips For Review entirely). Start with Categorize until you trust the rule.
10. Click **Save** to activate the rule.

The rule will apply to new transactions going forward and may also apply to existing transactions in your For Review tab.

> 💡 **Pro Tip:** Start by creating rules for your top 10 most frequent recurring vendors. Log into your bank statement and look for vendors that appear 4 or more times per month — those are your best rule candidates. You can set up all 10 rules in under 20 minutes, and they'll pay dividends every month going forward.

## Setting Rule Conditions

Conditions are the "if this" part of your rule. QuickBooks lets you match on three main criteria:

**Description contains / is exactly / doesn't contain**
This looks at the raw text from your bank feed. For example, your bank might describe every Comcast payment as "COMCAST CABLE 800-266-2278." A condition of "Description contains COMCAST" will match all variations of that text. Use "contains" for partial matches, "is exactly" only when you want an exact match (rarely needed), and "doesn't contain" to create exclusions.

**Amount is / is greater than / is less than / is between**
Use amount conditions to make rules more specific. For example, if you have two different Amazon Prime subscriptions and one is $14.99 while the other is $139.00 annually, you can create separate rules based on amount. Amount rules also work for payroll direct deposits, where the amounts are consistent.

**Bank text / Payee is**
If QuickBooks has already identified a payee from past transactions, you can set the condition to match that payee name rather than the raw description text.

You can combine multiple conditions using **AND** (all conditions must match) or **OR** (any condition triggers the rule). For most simple rules, a single condition on Description is sufficient.

**Example rule conditions:**
- Description contains "NETFLIX" → Category: Subscriptions, Payee: Netflix
- Description contains "SHELL" AND Amount is between $40.00 and $120.00 → Category: Auto: Fuel, Payee: Shell
- Description contains "GUSTO" → Category: Payroll Expenses, Payee: Gusto

## Assigning Category and Payee

The **Then** section of your rule determines how the transaction is recorded:

**Payee** — This is the vendor or customer name that will appear on the transaction. Set this to the clean, recognizable version of the name (e.g., "Netflix" instead of "NETFLIX.COM 866-579-7172"). If the payee doesn't exist yet in QuickBooks, you can create it directly from this field.

**Category** — Select the appropriate account from your chart of accounts. This is the same list you use when manually categorizing. Be specific: "Utilities: Internet" is better than just "Utilities."

**Class / Location** (optional) — If your business uses class tracking (e.g., separating revenue and expenses by department, job type, or location), you can assign a class here so the rule handles it automatically.

## Rule Priority and Conflict Resolution

If a transaction could match more than one rule, QuickBooks uses **priority order** to decide which rule wins. Rules higher in the list take precedence over rules lower in the list.

To see and adjust rule priority:
1. Go to **Banking > Rules**.
2. The rules are listed with a numbered order. The rule at the top (priority 1) takes precedence.
3. To change priority, hover over the left edge of any rule row — a drag handle will appear. Click and drag the rule up or down to reorder it.

**Example:** You might have a general rule that says "Description contains AMAZON → Office Supplies" and a more specific rule that says "Description contains AMAZON AND Amount is between $13.00 and $15.00 → Subscriptions" (for Amazon Prime). Put the specific rule above the general rule in priority order so the more precise match fires first.

> 💡 **Pro Tip:** If a rule is firing incorrectly on transactions you didn't intend it to match, don't delete the rule — edit it to add a more specific condition. For example, if your "contains APPLE" rule is catching both Apple.com purchases and Apple Orchard Farm Supply (a vendor), add an amount range or a second keyword to narrow it down.

## Editing and Deleting Rules

Rules need maintenance over time. A vendor might change their billing name, you might change bank accounts, or a subscription you canceled is no longer relevant.

**To edit a rule:**
1. Go to **Banking > Rules**.
2. Find the rule and click **Edit** (the pencil icon or the rule name).
3. Make your changes and click **Save**.

**To disable a rule temporarily:**
On the Rules list, toggle the rule's active/inactive switch. Inactive rules are kept in your list but don't fire on new transactions — useful for seasonal rules or rules you're unsure about.

**To delete a rule permanently:**
Click the dropdown arrow next to the Edit button and select **Delete**. Deleting a rule does not undo any transactions it already categorized.

## Common Mistakes to Avoid

- **Using "Automatically add to my books" too soon.** This is a powerful setting, but if your rule conditions aren't precise enough, it can silently miscategorize transactions without you noticing. Start with "Categorize" mode and watch for a few weeks before switching to auto-add.
- **Creating rules that are too broad.** A rule that matches "Description contains THE" will catch far too many unrelated transactions. Use specific keywords that uniquely identify the vendor.
- **Forgetting to update rules after vendor name changes.** Banks sometimes update how a vendor's name appears in your feed. If a rule stops firing, the vendor's description text may have changed — edit the rule condition accordingly.
- **Setting the wrong money direction.** A rule set to "Money out" will never fire on a deposit, even if all other conditions match. Double-check that you've selected the right direction before saving.
- **Never reviewing auto-added transactions.** Even with trusted rules, do a monthly spot-check of auto-categorized transactions in your Reviewed tab to make sure nothing slipped through incorrectly.

Identify three vendors that appear regularly in your bank feed — at least monthly. For each vendor, create a bank rule in **Banking > Rules > New Rule** with at least one condition (Description contains their name), assign the correct payee and category, and set the action to "Categorize the transaction." After saving all three rules, return to your For Review tab and see if any waiting transactions are now pre-categorized by the new rules. Approve them with a single click.

## Key Takeaways

- Bank rules automate the categorization of recurring transactions, dramatically reducing time spent in the For Review tab each week.
- Rules are built with conditions (if description contains X, or amount equals Y) and actions (assign this category and payee).
- Multiple conditions can be combined with AND/OR logic to create precise, reliable rules.
- Rule priority order determines which rule wins when a transaction matches more than one.
- Start rules in "Categorize" mode before switching to "Automatically add" — it lets you verify accuracy before giving the rule full autonomy.
